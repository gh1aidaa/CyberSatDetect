Terms of Use & Privacy Notice

Overview of how CyberSatDetect may be used and how telemetry and usage data are handled.

This section is written for an academic / research deployment of the platform.

Terms of Use

CyberSatDetect is intended for academic, research, and experimental analysis of satellite telemetry. By using this platform, you agree to operate it responsibly and within the scope of its design.

The system allows users to upload telemetry files, run anomaly detection, and review generated reports. It is not a certified operational safety tool, and all results must be interpreted in context.

  • Users must only upload valid telemetry files in the approved formats (\texttt{.csv} or \texttt{.npy}).
  • It is not permitted to intentionally upload malicious or irrelevant data.
  • Reverse engineering, bypassing controls, or tampering with the detection logic is not allowed.
  • Access credentials are personal and must not be shared with unauthorized parties.
  • The system may log basic actions (such as file uploads and detection runs) for traceability.

Legal Disclaimer

CyberSatDetect is developed as a graduation and research project. While it provides meaningful anomaly insights for telemetry streams, it is not guaranteed to detect every possible failure or cyber incident.

The developers and supervising institution are not liable for operational decisions made solely based on this platform. It is the responsibility of the user or mission operator to validate findings and integrate them with other monitoring tools and procedures.

Privacy Notice

The platform focuses on telemetry data, not on personal information. In the current implementation, processing is oriented around satellite channels, system health, and anomaly scores rather than user identity.

For a front-end prototype, uploaded files are handled locally in the browser environment, and no external transmission is performed unless explicitly configured in a backend environment.

  • Telemetry files are used exclusively for anomaly detection and model evaluation.
  • No personal data fields are required for basic use of the console.
  • File names and timestamps may be stored temporarily to support the user session and auditability.
  • Data is not shared with third parties in the context of this academic deployment.

Security Commitment

The system design follows a security-aware approach, aligned with the rest of CyberSatDetect. The web console and backend pipeline are structured to preserve data integrity, traceability, and model reliability.

  • File type and size are validated before processing to protect the pipeline.
  • Detection thresholds and internal model parameters are not exposed for direct manipulation.
  • Anomaly outputs shown in the UI are read-only, derived from a controlled processing path.
  • Key timestamps (such as last detection run) can be recorded for auditing and debugging.
  • The platform is intended to comply with baseline cybersecurity practices in an academic context.
