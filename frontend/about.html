About CyberSatDetect

An AI-driven anomaly detection system for satellite telemetry, built on hybrid LSTM–GRU models and secure processing pipelines.

This page provides a high-level overview of the project, its AI models, and the security mindset behind the design.

Core Models LSTM + GRU
Focus Satellite Telemetry
Mode Unsupervised + Continual

Project Overview

CyberSatDetect is a graduation project that targets the security and reliability of satellite operations by analyzing raw telemetry streams and automatically detecting abnormal behavior. The system ingests mission telemetry, cleans and normalizes it, segments the data into time windows, and then applies a hybrid deep-learning engine to score each window for anomalous patterns.

Instead of relying on static thresholds or manual inspection, CyberSatDetect uses a data-driven approach that learns temporal behavior directly from telemetry. This is particularly important for space missions, where complex subsystems, non-linear dynamics, and evolving operating modes make traditional rule-based monitoring difficult to maintain and scale.

The platform is structured as a secure, modular web console: the user uploads a telemetry file through the Data Upload page, runs the hybrid anomaly engine from the Anomaly Detection page, and then generates a mission-ready report from the Reports page. Each step is designed with both operational usability and security controls in mind.

Telemetry Workflow

The system follows a clear end-to-end pipeline. First, the user provides a telemetry file in an approved format (CSV or NPY). The data then passes through three main preprocessing stages:

  • Clean Data: remove corrupted rows, invalid values, and structural inconsistencies.
  • Normalization: scale and align telemetry channels to comparable ranges.
  • Time Segmentation: slice the stream into fixed-size windows that become anomaly segments.

After preprocessing, each segment is forwarded to the anomaly engine. The front-end console does not expose raw model parameters; instead, it visualizes the resulting scores and classifications in a controlled way. This separation keeps the user experience focused on operational insights while preserving model integrity.

Hybrid LSTM–GRU Anomaly Engine

At the core of CyberSatDetect is a hybrid anomaly engine that combines two recurrent neural architectures: a Long Short-Term Memory (LSTM) network and a Gated Recurrent Unit (GRU). Both models process the same telemetry segments but specialize in capturing different temporal behaviors. The LSTM focuses on longer-term dependencies, while the GRU reacts more quickly to short, sharp changes.

For each segment, both models produce an anomaly score. These scores are then fused into a single value using a weighted combination.

The fused score is compared against an internal threshold computed by the system, not by the operator, to decide whether a segment is considered anomalous. Selected anomalies can later be used to support continual learning, where the model is fine-tuned on evolving telemetry patterns without retraining from scratch.

Security Mindset

The design of the web console and backend pipeline follows a security-conscious mindset. Telemetry files are strictly validated before processing, anomaly thresholds and model internals are not exposed to direct user manipulation, and all outputs shown in the console are read-only views derived from a validated processing path.

Security-related aspects highlighted by this page include:

  • Safe ingestion of telemetry through strict file-type and size validation.
  • Model integrity by preventing end-users from changing thresholds or core detection logic.
  • Use of a documented, step-wise pipeline (Upload → Preprocess → Detect → Report) for traceability.
  • Support for auditability by exposing when a file was processed and when detection was last executed.

Overall, CyberSatDetect is not only an AI project but also a secure telemetry processing framework that combines anomaly detection, continual learning, and basic cyber controls tailored to satellite operations.
